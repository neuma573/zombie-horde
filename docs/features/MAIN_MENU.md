# 메인 메뉴와 캐릭터 선택

## 목적

- 게임 실행 직후 전투를 시작하지 않고 메인 메뉴를 표시한다.
- 설정 화면에서 전역 음소거 상태만 변경한다.
- 게임 시작 전에 두 외형 클래스 중 정확히 하나를 선택한다.
- `docs/references/male.png`, `docs/references/female.png`를 선택 카드에 표시한다.

## 화면 흐름

```text
메인 메뉴
  ├─ 설정 → 음소거/소리 듣기 → 메인 메뉴
  └─ 게임 시작 → 클래스 선택 → GameScene
```

- 클래스가 선택되기 전에는 전투를 시작할 수 없다.
- 클래스 선택은 단일 선택이며 새 선택이 이전 선택을 대체한다.
- 뒤로 가기는 메인 메뉴로 돌아간다.

## 전역 상태

Phaser 게임 전역 Registry에 다음 상태를 저장한다.

- `settings.soundEnabled`: 기본값 `true`
- `session.characterClassId`: 선택한 클래스 ID

현재 게임에는 오디오가 없으므로 `soundEnabled`는 재생 동작을 일으키지 않는다.
향후 오디오 구현은 이 값을 읽어 음소거 정책에 적용한다.

## 클래스

- `male-survivor`
- `female-survivor`

두 클래스는 현재 능력치와 게임 판정이 같다. 선택 화면만 서로 다른 크로키 이미지의
전신에 가까운 crop을 사용한다. 모바일은 전폭 가로 카드 두 장을 세로로 배치해 인물과
이름에 충분한 폭을 제공하고, 넓은 화면에서만 두 패널을 좌우에 배치한다. 하체를 포함한
전신 구도는 선택 전후에 유지한다. 선택한 패널은 `John Doe` 또는 `Jane Doe` 이름과 함께
전신 크기, 강조 색상 및 짧은 이동 효과를 키운다. 각 옵션은 `portraitTextureKey`,
`portraitUrl`, `portraitCrop`을 가지며 Vite가 참조 이미지를 빌드 에셋으로 포함한다.
URL이 없는 향후 옵션은 임시 실루엣으로 대체할 수 있다.

## 반응형과 수명 주기

- 메뉴는 `Phaser.Scale.RESIZE`와 CSS safe-area 값을 사용한다.
- resize 시 현재 화면과 선택·설정 상태를 유지하면서 UI만 다시 배치한다.
- Scene shutdown에서 resize 리스너와 메뉴 객체를 정리한다.
- PC와 모바일 모두 포인터 입력으로 조작할 수 있다.

## 제외

- 실제 오디오와 배경음악
- 클래스별 능력치와 스킬
- 캐릭터 외형을 Player 렌더링에 적용하는 작업
- 저장소 또는 브라우저 영구 저장

## 검증

- 음소거 토글은 기존 설정 객체를 변경하지 않고 새 상태를 반환한다.
- 두 클래스만 선택 가능하다.
- 새 클래스 선택은 이전 선택을 대체한다.
- 잘못된 클래스 ID는 선택 상태를 변경하지 않는다.
- `npm test`와 `npm run build`가 통과한다.
